# Проект как основа диплома

## Что лежит в монорепо

Текущий репозиторий — это монорепо `Nx + pnpm`, в котором уже собрана база для прикладной ВКР по разработке веб-платформы.

- `apps/cms` — backend на `Strapi 5`
- `apps/front` — frontend на `Astro 6`
- корневой `package.json` — orchestration-команды для совместной сборки и запуска

## Почему проект подходит для ВКР

Проект не ограничивается одной витринной страницей. В нем уже есть несколько функциональных подсистем:

- headless CMS для управления контентом;
- frontend-витрина для публикации материалов;
- публикация статей и проектов;
- модуль вакансий;
- форма отклика на вакансию;
- генерация типизированного API-клиента из OpenAPI.

Это позволяет формулировать тему ВКР не как "сделать сайт", а как разработку CMS-управляемой веб-платформы.

## Подтвержденные технические факты по коду

- Корневой `README.md` описывает монорепо из двух приложений: `apps/cms` и `apps/front`.
- Корневой `package.json` содержит команды `pnpm dev`, `pnpm build`, `pnpm generate:api` и `affected:*`.
- В `apps/cms` есть контентные сущности `article`, `project`, `vacancy`, `vacancy-application`, `industry`, `job-role`, `home-page`, `global`.
- В `apps/front` есть страницы и компоненты для `articles`, `projects`, `vacancies`.
- Во frontend используется `astro build`, при этом в конфигурации не задан server-adapter, что соответствует базовому сценарию предварительной сборки публичной витрины.
- Вакансии реализуют поиск, фильтрацию и пагинацию.
- Отклик на вакансию реализует валидацию, honeypot, rate limit и загрузку резюме.

## Возможная формулировка темы

`Разработка CMS-управляемой веб-платформы для маркетингового сайта и публикации вакансий`

## Что можно описывать в главах

### Аналитическая часть

- задачи маркетинговой веб-платформы;
- сравнение традиционного CMS-подхода и headless-подхода;
- выбор `Strapi` и `Astro`;
- аргументация в пользу предварительной сборки публичной витрины для контентных страниц;
- выбор монорепо и общего API-контракта.

### Проектная часть

- архитектура `frontend + CMS`;
- модель сущностей контента;
- модуль вакансий и откликов;
- генерация типизированного API;
- сценарии редактора и конечного пользователя.
