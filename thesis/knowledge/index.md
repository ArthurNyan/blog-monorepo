# Knowledge Base

Эта папка фиксирует все внешние требования, принятые допущения и опорные знания по дипломной работе.

## Что здесь лежит

- `requirements-herzen-vkr.md` — markdown-экстракт требований из файла РГПУ.
- `template-audit.md` — краткий технический аудит шаблона `edu-thesis-gost`.
- `compliance-diff.md` — расхождения между файлом требований и шаблоном преподавателя.
- `project-basis.md` — краткая выжимка по текущему монорепо как базе для ВКР.
- `diploma/` — рабочий knowledge hub по теме, целям, структуре, чекпоинтам и правилам для агента.
- `source-docs/vkr_pril_20_04_2022 (1).docx` — исходный файл требований без изменений.

Дополнительный исследовательский архив по референсным работам лежит в корне репозитория:

- `../../research/reference-theses/` — локальные копии похожих дипломов, карточки источников и сводный каталог.

## Источники истины

1. Основной рабочий стандарт оформления: шаблон преподавателя в `thesis/`.
2. Контрольный нормативный источник: `source-docs/vkr_pril_20_04_2022 (1).docx`.
3. Рабочая тема и содержание ВКР: текущий проект в корне монорепо.
4. Рабочие решения по содержанию диплома: `diploma/*.md`.

## Как работать с дипломом

1. Редактировать титульные данные в `../config/metadata.tex`.
2. Заполнять главы в `../content/*.tex`.
3. Вести список литературы в `../references.bib`.
4. Сверяться с `compliance-diff.md`, если шаблон расходится с формальными требованиями.
5. Для содержательных решений по теме и структуре сначала сверяться с `diploma/`.
6. Собирать документ из `thesis/` через `make build`, `make check`, `make ci`.

## Текущее решение по соответствию требованиям

- Шаблон преподавателя считается главным практическим стандартом.
- Формальные требования РГПУ сохранены отдельно и используются как контрольный чеклист.
- Найденные расхождения не правятся автоматически на первом проходе, а фиксируются для отдельного решения.
