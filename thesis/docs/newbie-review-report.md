# Code Review: Новички-first (production-strict)

Дата: 14.03.2026  
Охват: весь репозиторий (`Makefile`, `gostvkr-lua.cls`, `README`, `docs`, `content`).

## Findings

### High
1. Смешанный язык пользовательского DX (RU/EN) в пайплайне сборки.
   - Влияние: новичок получает неоднородные подсказки, сложнее локализовать причину ошибки.
   - Место: `Makefile` (помощь и сообщения проверок, ранее строки 30-193).
   - Рекомендация: полная русификация help и диагностик при сохранении имен целей.
   - Статус: исправлено.

2. Недостаточная документация ядра класса для первого входа.
   - Влияние: сложно понять границу между публичным API и внутренними макросами.
   - Место: `gostvkr-lua.cls` (блоки опций, шрифтов, титула и TOC).
   - Рекомендация: добавить структурные комментарии в `.cls` и отдельный короткий гайд.
   - Статус: исправлено (`docs/cls-guide.md` + комментарии в классе).

### Medium
1. README описывал проект недостаточно маршрутно для новичка.
   - Влияние: старт возможен, но без четкой последовательности «что открыть/изменить/проверить».
   - Место: `README.md`.
   - Рекомендация: формат «старт за 5 минут» + явные зоны редактирования + быстрые решения типовых ошибок.
   - Статус: исправлено.

2. Учебные главы содержали примеры, но без явной навигации от простого к сложному.
   - Влияние: при первом чтении сложнее понять порядок освоения окружений.
   - Место: `content/02-chapter-1.tex`, `content/03-chapter-2.tex`.
   - Рекомендация: короткие вводные абзацы-навигация в начале глав.
   - Статус: исправлено.

## Roadmap (3 волны)

### Quick wins
- Полностью русифицировать `make help` и сообщения `lint/check/fontcheck`.
- Перестроить README в формат «маршрут новичка».
- Добавить вводные навигационные абзацы в главы.

Критерий готовности:
- `make help` и логи проверок читаются полностью на русском.
- Новичок может пройти путь `metadata -> content -> make build -> make ci` без внешних пояснений.

Проверка:
- `make help`
- `make build`
- `make check`

### Structural cleanup
- Добавить структурные комментарии в `gostvkr-lua.cls`.
- Зафиксировать краткий гид по классу в `docs/cls-guide.md`.
- Синхронизировать термины между README, методичкой и классом.

Критерий готовности:
- Публичный API и внутренний API класса отделены явно.
- Нет противоречий по названиям команд/опций/файлов в документации.

Проверка:
- Ручная сверка `README.md`, `docs/cls-guide.md`, `gostvkr-lua.cls`.

### Hardening
- Зафиксировать русскоязычные fail-fast сообщения по шрифтам и активам.
- Сохранить регресс-контроль в `make check` и `make fontcheck`.
- Поддерживать полный прогон `make ci` как критерий готовности изменений.

Критерий готовности:
- Негативные тесты шрифтов падают с понятной причиной.
- Позитивные сценарии сборки и проверки воспроизводимы.

Проверка:
- `make distclean && make build`
- повторный `make build`
- `make fontcheck`
- `make ci`
