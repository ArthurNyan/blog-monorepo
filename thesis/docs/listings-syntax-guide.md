# Справочник синтаксиса для новичка

Эта методичка дополняет основной PDF-текст ВКР и объясняет синтаксис используемых
конструкций без перегрузки основной работы учебными комментариями.

## 0. Перед началом
- Для маршрута сборки и проверки используйте `README.md`.
- Для устройства класса используйте `docs/cls-guide.md`.
- Основной принцип: сначала собирается рабочий текст, затем добавляются сложные окружения.

## 1. Как читать демо-проект
- Сначала прочитайте `content/01-introduction.tex`.
- Затем идите по порядку:
  - `content/02-chapter-1.tex` — базовые и средние конструкции.
  - `content/03-chapter-2.tex` — advanced-окружения.
  - `content/05..08-appendix-*.tex` — готовые шаблоны для копирования.

## 2. Формулы
### Что делает конструкция
- `equation` — одна нумеруемая формула.
- `align` — несколько строк с выравниванием по `&`.
- `cases` — кусочно-заданная функция.

### Минимальный синтаксис
```tex
\begin{equation}
  y = kx + b,
  \label{eq:sample}
\end{equation}
```

### Типичные ошибки
- Нет `\label{...}` внутри окружения.
- Ссылка `\eqref{...}` указывает на несуществующий label.
- Разные префиксы меток в одном проекте.

### Правильный шаблон
- Для формул используйте префикс `eq:`.
- Подписывайте сразу после формулы, а ссылку делайте в следующем абзаце.

## 3. Рисунки
### Что делает конструкция
- `figure` — плавающее окружение рисунка.
- `subfigure` (из `subcaption`) — подрисунки внутри одного общего рисунка.

### Минимальный синтаксис
```tex
\begin{figure}[tbp]
  \centering
  \includegraphics[width=0.75\linewidth]{assets/images/fig-01-pastebin-architecture.png}
  \caption{Пример рисунка}
  \label{fig:sample}
\end{figure}
```

### Типичные ошибки
- `\label` стоит до `\caption`.
- Файл изображения лежит не по пути `assets/images/...`.
- Подпись отсутствует или не совпадает с контекстом.

### Правильный шаблон
- Сначала `\caption`, потом `\label`.
- Для рисунков используйте префикс `fig:`.

## 4. Таблицы
### Что делает конструкция
- `tabular` + `booktabs` — чистые линии и аккуратная структура таблиц.
- `longtable` — таблица переносится на следующую страницу автоматически.
- `siunitx` — выравнивание чисел по формату.

### Минимальный синтаксис
```tex
\begin{table}[tbp]
  \centering
  \caption{Контрольные метрики}
  \label{tab:sample}
  \begin{tabular}{@{}lS[table-format=2.2]@{}}
    \toprule
    Метрика & {Значение} \\
    \midrule
    Accuracy & 95.20 \\
    \bottomrule
  \end{tabular}
\end{table}
```

### Типичные ошибки
- Использование вертикальных линий вместо `booktabs`.
- Слишком длинные заголовки без переноса (`\makecell` не применен).
- Нет ссылки на таблицу в тексте.

### Правильный шаблон
- Для таблиц используйте префикс `tab:`.
- Перед таблицей или сразу после нее дайте ссылку вида `таблица~\ref{tab:...}`.

## 5. Листинги кода
### Что делает конструкция
- `listings` вставляет код без shell-escape.
- В проекте используется минимальная настройка листингов с локальным шрифтом `Fira Code`.
- Подпись листинга ведет себя как обычная подпись объекта.

### Как выбирать язык
- По умолчанию `lstlisting` использует язык Python.
- Для LaTeX-фрагментов задавайте локально `language={[LaTeX]TeX}`.
- Для команд shell/Makefile задавайте локально `language=bash`.

### Минимальный синтаксис
```tex
\begin{lstlisting}[caption={Пример кода},label={lst:sample}]
def normalize(value, low, high):
    if high == low:
        return 0.0
    return (value - low) / (high - low)
\end{lstlisting}
```

```tex
\begin{lstlisting}[language={[LaTeX]TeX},texcsstyle=*\color{lstKeyword},alsoletter={\\},morekeywords={begin,end,chapter,section,subsection,label,ref,eqref,caption,includegraphics},caption={Пример LaTeX-фрагмента},label={lst:sample-latex}]
\begin{equation}
  y = kx + b,
  \label{eq:line}
\end{equation}
\end{lstlisting}
```

```tex
\begin{lstlisting}[language=bash,caption={Пример команды сборки},label={lst:sample-bash}]
make build
\end{lstlisting}
```

### Типичные ошибки
- В коде используются нестабильные Unicode-символы.
- Нет подписи и метки `lst:...`.
- Для обычных Python-листингов вручную задается `language=Python`, хотя он уже является глобальным дефолтом.
- Отсутствуют файлы `FiraCode-*.ttf` в `fonts/fira-code/`.

### Правильный шаблон
- Для листингов используйте префикс `lst:`.
- Держите код компактным, но рабочим и понятным.
- Для основных Python-листингов не указывайте `style` и `language`.
- Для LaTeX/Bash добавляйте только локальный параметр `language`.

## 6. Алгоритмы
### Что делает конструкция
- `algorithm` создает плавающий объект с подписью.
- `algpseudocode` оформляет шаги псевдокода.

### Минимальный синтаксис
```tex
\begin{algorithm}[tbp]
  \caption{Пример алгоритма}
  \label{alg:sample}
  \begin{algorithmic}[1]
    \Require Входной массив data
    \Ensure Отсортированный массив
    \ForAll{элемент в data}
      \State Обновить состояние
    \EndFor
    \State \Return result
  \end{algorithmic}
\end{algorithm}
```

### Типичные ошибки
- `\label` забывают после `\caption`.
- В тексте нет ссылки на алгоритм.
- Смешиваются стили псевдокода.

### Правильный шаблон
- Для алгоритмов используйте префикс `alg:`.
- Один алгоритм = одна четкая задача.

## 7. Перекрестные ссылки
### Минимальные правила
- Ставьте `\label` в том же окружении, где создается номер.
- Используйте единый стиль префиксов:
  - `eq:`, `fig:`, `tab:`, `lst:`, `alg:`.
- После добавления новых ссылок запускайте сборку минимум 2 раза (`make build`).

## 8. Мини-чеклист новичка
- Все важные объекты подписаны и имеют `\label`.
- На каждый объект есть хотя бы одна ссылка из текста.
- `make build` и `make check` проходят без ошибок.
