# agent.md

Инструкция для работы с LaTeX-шаблоном ВКР в этой папке.

## Назначение

`thesis/` — изолированная рабочая зона диплома. Она не зависит от `apps/front` и `apps/cms` во время сборки PDF.

## Что редактировать

- `config/metadata.tex` — титульные данные.
- `content/01-introduction.tex` — введение.
- `content/02-chapter-1.tex` — аналитическая глава.
- `content/03-chapter-2.tex` — проектная глава.
- `content/04-conclusion.tex` — заключение.
- `content/05-appendix-a.tex` ... `content/08-appendix-d.tex` — приложения.
- `references.bib` — список литературы.

## Что не трогать без необходимости

- `gostvkr-lua.cls` — ядро класса.
- `main.tex` — точка сборки и базовая композиция документа.
- `Makefile` — пайплайн сборки и проверок.
- `fonts/` — локальные шрифты шаблона.

## Команды

Запускать из папки `thesis/`.

```bash
make build
make check
make ci
make clean
```

## Как устроена сборка на этой машине

- `Makefile` уже настроен на локальный `TinyTeX` из `~/Library/TinyTeX/bin/universal-darwin`.
- `pdftotext` и `pdffonts` берутся из `/opt/homebrew/bin`.
- Поэтому обычный `make build` должен работать без ручного `export PATH`.

## Порядок безопасной работы

1. Менять сначала `metadata.tex` и текст `content/*.tex`.
2. После смысловых правок запускать `make build`.
3. Перед фиксацией состояния запускать `make check`.
4. Перед сдачей или большим merge запускать `make ci`.

## Важные ограничения шаблона

- Шаблон рассчитан на `LuaLaTeX`, не на `pdflatex`.
- Библиография идет через `biber`.
- `make check` ожидает, что лог сборки чистый.
- `make ci` сейчас проходит, но `chktex` выдает предупреждения на учебном sample-контенте. Это не блокирует сборку, но при переписывании глав лучше постепенно убирать такие предупреждения.

## Практические замечания

- Не запускай несколько `make build`/`make check`/`make ci` параллельно в одной и той же папке: у LaTeX есть гонки по `.aux`, `.toc`, `.run.xml`.
- Если сборка начала вести себя странно после неудачного прогона, сначала сделай:

```bash
make clean
make build
```

- Если менялись библиография, ссылки, оглавление или длинные таблицы, повторный `make build` после первого прогона нормален.
