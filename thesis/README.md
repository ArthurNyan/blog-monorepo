# Шаблон ВКР на LuaLaTeX

Проект предназначен для подготовки ВКР в формате «образец + учебные примеры».  
Основной принцип: в PDF остается академический стиль, а синтаксис и пошаговые подсказки вынесены в документацию.

## Старт за 5 минут
1. Откройте `config/metadata.tex` и заполните данные титульного листа.
2. Отредактируйте текст в `content/*.tex`.
3. Проверьте источники в `references.bib`.
4. Выполните:
   ```bash
   make build
   ```
5. Для полной проверки выполните:
   ```bash
   make ci
   ```

## Что редактировать новичку в первую очередь
- `config/metadata.tex` — тема, ФИО, руководитель, город, год.
- `content/*.tex` — введение, главы, заключение, приложения.
- `references.bib` — библиография.

## Что обычно не нужно менять
- `gostvkr-lua.cls` — ядро класса и типографика.
- `Makefile` — пайплайн сборки и проверки.

Если нужно понять устройство класса, откройте `docs/cls-guide.md`.

## Карта проекта
- `gostvkr-lua.cls` — класс документа (`extbook`, главы, титул, оглавление, библиография).
- `main.tex` — точка входа, подключение пакетов, порядок частей документа.
- `config/metadata.tex` — значения для `\gostsetup{...}`.
- `content/01..08-*.tex` — текст ВКР и учебные примеры.
- `assets/images/fig-*.png` — изображения для разделов.
- `assets/title-2025-logo.png` — логотип для титульной страницы.
- `fonts/` — PT Astra Serif (основной текст).
- `fonts/fira-code/` — Fira Code (листинги кода).
- `docs/listings-syntax-guide.md` — методичка по формулам, таблицам, листингам, алгоритмам.
- `docs/assets-sources.md` — источники и лицензии изображений.
- `docs/newbie-review-report.md` — итог production-strict ревью по профилю «новички-first».

## Зафиксированные правила оформления
- Класс: `extbook`.
- Заголовки глав: `Глава N. Название`.
- Оглавление: отдельная страница, заголовок `Оглавление` по центру.
- Введение и заключение: ненумеруемые крупные разделы (с добавлением в оглавление).
- Нумеруемых глав основной части: ровно две (`Глава 1` и `Глава 2`).
- Приложения: блок `ПРИЛОЖЕНИЯ` и `Приложение А` на одной странице; `Приложение Б/В/Г` — с новой страницы.
- Библиография: `Список литературы`.
- URL: стиль основного текста (`\urlstyle{same}`).
- Иерархия внутри главы: `\section + \subsection`.

## Публичный API класса
- Опции `\documentclass`:
  - `degree=bachelor|master`
  - `language=ru-en|ru`
  - `bibstyle=gost-numeric` (или другой стиль `biblatex-gost`)
- Команды:
  - `\gostsetup{...}`
  - `\maketitlepage`
  - `\frontchapter{...}`
  - `\makeintroduction`
  - `\makeconclusion`
  - `\printgostbibliography`
  - `\makeappendicesblock`
  - `\appendixchapterfirst{...}{...}`
  - `\appendixchapternext{...}{...}`
  - `\makeabstractru{...}` и `\makeabstracten{...}` (опционально)

## Команды Makefile
- `make help` — справка по целям.
- `make build` — сборка `main.pdf`.
- `make rebuild` — полная пересборка.
- `make watch` — непрерывная сборка.
- `make lint` — статическая проверка `chktex + lacheck`.
- `make check` — проверка лога, маркеров PDF и контрактов ассетов.
- `make fontcheck` — позитивный и негативный тесты шрифтов.
- `make ci` — полный цикл (`build + lint + check + fontcheck`).
- `make clean` — удалить промежуточные файлы.
- `make distclean` — удалить все артефакты, включая PDF и отчеты.

## Частые ошибки и быстрые решения
- Ошибка про PT Astra Serif:
  - проверьте 4 файла `pt-astra-serif_*.ttf` в `fonts/`.
- Ошибка про Fira Code:
  - проверьте `FiraCode-*.ttf` в `fonts/fira-code/`.
- Ошибка по логотипу титула:
  - проверьте путь `logopath` в `config/metadata.tex`.
- Ошибки ссылок/оглавления:
  - запустите `make build` дважды.

## Как проверять себя перед сдачей
1. `make distclean && make build`
2. Повторно `make build` (идемпотентность)
3. `make check`
4. `make ci`

Если все команды проходят, техническая часть шаблона готова.
